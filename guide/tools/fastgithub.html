GitHub访问慢打不开？快使用加速神器FastGithub

Github 作为全球最大的开源社区和代码托管平台，蕴藏着非常多的宝藏项目，即使不是程序员，也应该熟练掌握其使用技巧，学会找到解决问题之道。

FastGithub 是一个 Github 加速神器，可以解决 github 打不开、用户头像无法加载、releases 无法上传下载等问题。

目前最新版本 2.1.4。请参照对应章节下载并运行。

Windows-x64

下载

https://ghproxy.com/https://github.com/dotnetcore/FastGithub/releases/download/2.1.4/fastgithub_win-x64.zip

运行

首先解压缩 fastgithub_win-x64.zip ；

进入解压后的文件夹，双击运行 FastGithub.UI.exe 。

此时再打开 github.com 应该已经可以正常访问了，若有问题请查看软件内 问答 板块。

使用完请及时关闭。

macOS

下载

https://ghproxy.com/https://github.com//dotnetcore/FastGithub/releases/download/2.1.4/fastgithub_osx-x64.zip

ARM 版本（m1 系列）

https://ghproxy.com/https://github.com/dotnetcore/FastGithub/releases/download/2.1.4/fastgithub_osx-arm64.zip

解决 “XXX” cannot be opened because the developer cannot be verified

打开终端进入 FastGithub 执行文件所在路径执行命令：

双击运行 FastGithub 并安装证书。

打开 FastGithub 后，目录内会生成 cacert 目录，双击打开 fastgithub.cer ，系统弹出 Keychain Access 窗口，列表中双击 FastGitHub，弹出证书详情窗口，展开 Trust 并选择 Always Trust 。

打开 mac 设置，网络，点击高级，选择代理，勾选 自动代理配置，填写 FastGithub 窗口提示的地址 http://127.0.0.1:38457 。

使用完毕后请手动复原。

Linux-x64

下载

https://ghproxy.com/https://github.com/dotnetcore/FastGithub/releases/download/2.1.4/fastgithub_linux-x64.zip

ARM 版本：

https://ghproxy.com/https://github.com/dotnetcore/FastGithub/releases/download/2.1.4/fastgithub_linux-arm64.zip

运行

终端直接运行 sudo ./fastgithub

或配置为服务

最后设置系统自动代理为 http://127.0.0.1:38457 ，或手动代理 http/https为127.0.0.1:38457 。

使用完毕后请手动复原。
